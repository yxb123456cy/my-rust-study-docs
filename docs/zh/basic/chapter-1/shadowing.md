# 变量遮蔽 (Shadowing)

在 Rust 中，你可以声明一个与之前变量同名的新变量。Rust 社区称之为**变量遮蔽（Shadowing）**。这意味着第一个变量被第二个变量遮蔽了，当我们使用这个变量的名称时，编译器看到的是第二个变量（直到该变量的作用域结束或被再次遮蔽）。

## 如何使用遮蔽

你可以通过使用相同的变量名并重复使用 `let` 关键字来遮蔽一个变量：

```rust
fn main() {
    let x = 5;

    // 遮蔽了前面的 x，现在 x 的值是 6
    let x = x + 1;

    {
        // 在内部作用域中再次遮蔽
        let x = x * 2;
        println!("内部作用域中 x 的值为: {}", x); // 输出 12
    }

    // 离开内部作用域后，内部的遮蔽失效，恢复到外部作用域的 x
    println!("外部作用域中 x 的值为: {}", x); // 输出 6
}
```

## 遮蔽 (Shadowing) 与 可变变量 (mut) 的区别

虽然遮蔽和可变变量（使用 `mut`）看起来都是为了改变变量的值，但它们有着本质的区别，主要体现在以下两个方面：

### 1. 遮蔽本质上是创建了新变量
使用 `let` 遮蔽变量时，实际上是**创建了一个全新的同名变量**。因此，我们可以对该值执行一些操作，然后在操作完成后，这个新变量默认仍然是不可变的（除非你使用了 `let mut`）。
而如果我们使用了 `mut` 重新赋值，仅仅是修改了原有变量内存空间中的数据，并没有创建新变量。

### 2. 遮蔽允许改变变量的类型
因为遮蔽是重新使用 `let` 关键字创建了一个新变量，所以**我们可以改变该值的类型，同时复用同一个变量名**。这在进行数据类型转换或解析时非常方便。

**使用遮蔽改变类型的例子（合法）：**
```rust
fn main() {
    // spaces 最初是一个字符串切片类型
    let spaces = "   ";
    
    // 遮蔽 spaces，将其类型变为 usize（无符号整数），代表空格的数量
    let spaces = spaces.len();

    println!("空格的数量是: {}", spaces);
}
```

**尝试使用 `mut` 改变类型（会报错）：**
```rust
fn main() {
    let mut spaces = "   ";
    // 错误：不能改变变量的类型
    // spaces = spaces.len(); 
}
```
编译器会提示 `mismatched types` (类型不匹配)，因为 `mut` 只能改变值，不能改变变量在最初声明时确定的类型（Rust 是强类型静态语言）。

## 总结
- **变量遮蔽**通过重新使用 `let` 关键字实现。
- 它允许你在保持数据不可变性的同时，对值进行一系列的连续转换。
- 它允许你复用变量名，即使在转换过程中数据的类型发生了变化，从而避免了去起诸如 `spaces_str` 和 `spaces_num` 这样繁琐的变量名。
