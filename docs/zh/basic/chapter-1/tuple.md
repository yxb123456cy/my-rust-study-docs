# Rust的复合类型 (一)：元组 (Tuple)

在了解了标量类型之后，我们来看看**复合类型 (Compound Types)**。复合类型可以将多个值组合成一个类型。Rust 有两个原生的复合类型：**元组 (Tuple)** 和 **数组 (Array)**。

本篇文档将详细介绍元组。

## 1. 什么是元组？

元组是一个将**多种不同类型**的值组合进一个复合类型的主要方式。元组有两个重要特性：
1. **长度固定**：一旦声明，其长度不会增大或缩小。
2. **类型可异**：元组内的每一个位置都有一个类型，而且这些不同位置的类型**不需要是相同的**。

## 2. 定义元组

我们通过编写包含在圆括号 `()` 中的逗号分隔的值列表来创建一个元组。

```rust
fn main() {
    // 自动推断类型
    let tup1 = (500, 6.4, 1);
    
    // 显式指定类型
    let tup2: (i32, f64, u8) = (500, 6.4, 1);
}
```

在上面的例子中，`tup2` 绑定到了整个元组，因为元组被认作是单个复合元素。

## 3. 访问元组中的值

想要从元组中获取个别的值，Rust 提供了两种主要的方法：**模式匹配解构** 和 **点号索引**。

### 方法一：模式匹配解构 (Destructuring)

我们可以使用模式匹配（Pattern Matching）来解构（Destructure）一个元组值，这类似于 JavaScript 中的解构赋值。

```rust
fn main() {
    let tup = (500, 6.4, 1);

    // 将元组的值分别绑定到 x, y, z 三个变量上
    let (x, y, z) = tup;

    println!("y 的值是: {}", y); // 输出: 6.4
}
```
在这个例子中，程序首先创建了一个元组，接着使用 `let` 和一个模式将元组拆分成了三个单独的变量 `x`、`y` 和 `z`。

### 方法二：点号索引 (Dot Indexing)

除了通过模式匹配解构，我们还可以使用 `.` (点号) 加上要访问的值的**索引**来直接访问它们。
注意：**元组的索引是从 0 开始的**，这与大多数编程语言中的数组一样。

```rust
fn main() {
    let x: (i32, f64, u8) = (500, 6.4, 1);

    let five_hundred = x.0; // 访问第一个元素
    let six_point_four = x.1; // 访问第二个元素
    let one = x.2; // 访问第三个元素

    println!("第一个值是: {}", five_hundred);
}
```

## 4. 单元类型 (Unit Type)

有一种没有任何值的特殊元组，它写成 `()`。
这个类型被称为 **单元类型 (Unit Type)**，而这个值被称为 **单元值 (Unit Value)**。

- 如果一个表达式不返回任何其他值，它通常会隐式地返回单元值。例如，那些不返回任何有意义值的函数（在其他语言中可能是 `void`），在 Rust 中实际上是返回了 `()`。

```rust
fn main() {
    let empty_tuple: () = ();
}
```
