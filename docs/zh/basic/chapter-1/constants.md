# 使用 const 定义常量

除了不可变变量，Rust 还有另一种不需要改变的值：**常量（Constants）**。常量就像是不可变变量，但常量有着更严格的限制。

## 定义常量

在 Rust 中，我们使用 `const` 关键字来定义常量，而不是 `let`。定义常量时有以下几个规则必须遵守：

1. 必须始终显式地标注数据类型（例如 `u32`）。
2. 常量可以在任何作用域中声明，包括全局作用域。
3. 常量只能被绑定到**常量表达式**，而不能绑定到函数调用的结果，或任何只能在运行时计算出的值。

### 示例

```rust
// 定义一个全局常量
const THREE_HOURS_IN_SECONDS: u32 = 60 * 60 * 3;

fn main() {
    println!("Three hours in seconds is: {}", THREE_HOURS_IN_SECONDS);
}
```

## 常量的命名规范

Rust 常量的命名规范是：**全部大写，并使用下划线分隔单词（SCREAMING_SNAKE_CASE）**。编译器会对不符合规范的常量命名发出警告。

## 常量与不可变变量（let）的区别

虽然常量和不可变变量都不允许改变值，但它们之间存在一些关键区别：

- **关键字**：常量使用 `const` 声明，不可变变量使用 `let` 声明。
- **类型标注**：常量**必须**显式指定类型，而 `let` 变量通常可以依靠编译器的类型推导。
- **作用域**：常量可以在任何作用域（包括全局）定义，而 `let` 不能在全局作用域中直接使用。
- **赋值限制**：常量只能被赋值为常量表达式（即编译期就能确定的值），而 `let` 变量可以被赋值为在运行时计算出的值（比如函数返回值）。
- **不可变性**：常量永远不可变，没有 `const mut` 这种写法。

在编写 Rust 代码时，对于程序中硬编码且多处使用的值（如超时时间、最大重试次数等），使用常量是一个非常好的习惯，它能让代码更易于维护和理解。
