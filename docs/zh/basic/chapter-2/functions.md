# Rust中的函数详解

在 Rust 中，**函数 (Functions)** 是无处不在的。你已经见过语言中最重要的函数之一：`main` 函数，它是很多程序的入口点。你也见过 `fn` 关键字，它用来声明新函数。

Rust 代码中的函数和变量名使用 **snake_case** 规范风格。在 snake_case 中，所有字母都是小写并使用下划线分隔单词。

## 1. 定义与调用函数

我们在 Rust 中通过输入 `fn`，后跟着函数名和一对圆括号来定义函数。大括号告诉编译器哪里是函数体的开始和结尾。

```rust
fn main() {
    println!("Hello, world!");

    // 调用另一个函数
    another_function();
}

fn another_function() {
    println!("Another function.");
}
```

**注意**：在 Rust 中，定义函数的位置并不重要。我们把 `another_function` 定义在了 `main` 之后，但把它定义在 `main` 之前也是完全合法的。Rust 并不关心函数定义在哪里，只要定义了并且在作用域内就可以被调用。

## 2. 函数参数 (Parameters)

函数可以被定义为拥有**参数 (parameters)**，这是一种特殊的变量，是函数签名的一部分。当函数拥有参数（形参）时，你可以为这些参数提供具体的值（实参）。

在函数签名中，**必须**声明每个参数的类型。这是 Rust 设计中一个经过慎重考虑的决定：要求在函数定义中提供类型注解，意味着编译器几乎从不需要你在代码的其他地方注明类型来指出你指的是什么类型。

```rust
fn main() {
    // 传递实参 5 给函数
    print_number(5);
    
    // 传递多个实参
    print_labeled_measurement(5, 'h');
}

// 定义带有一个参数的函数
fn print_number(x: i32) {
    println!("x 的值是: {}", x);
}

// 定义带有多个参数的函数，参数之间用逗号分隔
fn print_labeled_measurement(value: i32, unit_label: char) {
    println!("测量值是: {}{}", value, unit_label);
}
```

## 3. 语句和表达式 (Statements and Expressions)

函数体由一系列语句组成，也可选地以表达式结束。因为 Rust 是一门基于表达式的语言，这在 Rust 中是一个需要理解的重要概念。

*   **语句 (Statements)** 是执行一些操作但不返回值的指令。
*   **表达式 (Expressions)** 计算并产生一个值。

```rust
fn main() {
    // 这是一个语句：它执行了绑定操作，但不返回值
    let y = 6; 

    // let x = (let y = 6); // 错误！语句没有返回值，不能赋给 x
    
    // 这是一个包含表达式的块，块本身也是一个表达式
    let y = {
        let x = 3;
        x + 1 // 注意：这里没有分号！这是一个表达式，返回 4
    };

    println!("y 的值是: {}", y);
}
```
**关键区别**：表达式的结尾没有分号。如果你在表达式的结尾加上分号，它就变成了语句，从而不再返回一个值。在探索函数返回值时请牢记这一点。

## 4. 具有返回值的函数

函数可以向调用它的代码返回值。我们不在函数命名中指定返回值的名称，但我们**必须**在箭头 `->` 之后声明它的类型。

在 Rust 中，函数的返回值等同于函数体中最后一个表达式的值。
*   你可以使用 `return` 关键字并指定一个值来从函数中提前返回。
*   但大多数函数隐式地返回最后一个表达式的值。

```rust
fn five() -> i32 {
    5 // 这是一个表达式，返回 5。注意没有分号！
}

fn plus_one(x: i32) -> i32 {
    x + 1 // 如果加了分号 `x + 1;`，它就会变成语句，返回单元类型 `()`，进而导致编译错误
}

fn main() {
    let x = five();
    println!("x 的值是: {}", x); // 输出 5

    let y = plus_one(5);
    println!("y 的值是: {}", y); // 输出 6
}
```

### 为什么不能加分号？
如果你在 `plus_one` 的末尾加上了分号：

```rust
// 这是一个会报错的代码
fn plus_one(x: i32) -> i32 {
    x + 1; // 错误：加上分号后变成了语句，没有返回 i32 而是返回了 `()`
}
```
编译器会提示一个 `mismatched types`（类型不匹配）的错误。它期望返回 `i32`，但实际上因为有分号，语句返回了 `()`（单元类型）。这是 Rust 初学者最容易犯的错误之一！

---

**小结**：
- 使用 `fn` 关键字定义函数。
- 参数必须标注类型。
- Rust 是基于表达式的语言：区分好带有分号的**语句**和不带分号的**表达式**。
- 函数默认返回最后一个表达式的值，提前返回可以使用 `return`。
